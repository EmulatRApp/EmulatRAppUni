Virtual Disk Images

An emulated fixed disk is backed by a plain raw, flat image file: the byte at offset LBA × sector-size is the first byte of that logical block. The ATA fixed disk (device dqa0) uses 512-byte sectors and is read-write, so an operating-system installer can write to it and the writes persist.

 

Images are created from the geometry catalog -- automatically at boot via create_if_missing, or manually with tools/mkdisk.py (see Create Guest Disk Storage). The file is bare: no partition table or DEC disklabel until the guest install writes them.

 

Attaching an image

A manifest storage entry names the channel, unit, type, and the media file. With type = ata_disk the image is opened as a 512-byte read-write block device on the CY82C693 IDE channel and appears to the firmware as dqa0. On launch the boot log confirms the attach:

 

Storage: attached ATA disk 'dqa0.img' to IDE ch0 unit0

 

Note: a blank image is not bootable on its own -- an operating system is installed onto it (booted from an ISO on the optical device). Enumeration of a bootable disk is in progress; see Current Development Status.
