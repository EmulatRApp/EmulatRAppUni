Terminal Clients

Connection steps for the common clients. In every case the target is localhost port 10023 (or a remote host and the port the instance was started with), using a RAW connection.

 

Plink

plink -raw localhost -P 10023 # local

plink -raw 192.168.1.50 -P 10023 # remote

plink -v -raw localhost -P 10023 -log emulator.log # verbose + session log

Useful arguments: -raw (raw TCP stream), -P <port> (TCP port), -v (verbose diagnostics), -log <file> (save session log).

 

PuTTY

Launch PuTTY, set Host Name localhost and Port 10023, choose Connection type Raw, then press Open. For a remote instance use its IP in place of localhost.

 

Tera Term

New connection, select TCP/IP, Host localhost, TCP Port 10023, then connect.

 

MobaXterm

Session, then a Network session: Remote host localhost, Port 10023, Protocol TCP, then connect.

 

Netcat (Linux / macOS)

nc localhost 10023 # local

nc 192.168.1.50 10023 # remote

Useful for scripting and automated testing.

 

Telnet

Some telnet clients connect (telnet localhost 10023), but because EmulatR uses a raw socket and performs no Telnet option negotiation, a raw-capable client (PuTTY Raw, Plink -raw, Tera Term TCP/IP, MobaXterm TCP) is preferred.

 

Verifying the connection

After connecting you should see console output -- a banner, the >>> prompt, or an in-progress power-up sequence -- depending on the current emulator state.

 

Troubleshooting

Connection refused: verify EmulatR is running, the console server is enabled, and the port number is correct.

Connection timeout: verify firewall configuration and the host address, and that the listener is active.

No console output: verify the boot has progressed, console output is enabled, and you are on the right port.
