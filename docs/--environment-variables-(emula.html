Environment Variables

The runtime environment variables EmulatR reads at launch, plus the compile-time flags that look like env vars but are CMake options. All values are read with std::getenv -- set them in the launching shell before starting the process (inline bash assignment is the normal form).

 

Three gotchas

- Presence, not value. Most gates test only that the variable is set, so VAR=0 still ENABLES them. Exceptions that parse a value are noted below.

- Read once. The trace / diagnostic gates cache their result on first call; exporting them mid-run has no effect.

- Paths are CWD-relative. The stop sentinel and the snapshots directory resolve against the launch directory; launch from the Emulatr root.

 

Snapshot / autoload

Variable

Default

Effect

EMULATR_CONSOLE_SNAPSHOT

off

Arms the console-marker snapshot watch; a matching input line writes a .axpsnap.

EMULATR_SNAPSHOT_MARKER

set oem_string snapshot

The exact line that triggers a console snapshot (internal whitespace NOT normalized).

EMULATR_AUTOSNAP

on

=off disables periodic auto saves. Mirrors --autosnapshot off.

EMULATR_NO_AUTOLOAD

unset

Skip autoload-newest; force a genuine cold boot. Mirrors --no-autoload.

EMULATR_SNAPSHOT_PRUNE

unset

Enable pruning of old snapshots (off by default).

 

Process / platform

Variable

Default

Effect

EMULATR_STOP_FILE

EMULATR_STOP

Path of the graceful-stop sentinel; touch it to flush flash NVRAM and exit cleanly.

EMULATR_NO_PUTTY

unset

Do not auto-launch PuTTY; the listen socket still comes up.

EMULATR_CONSOLE_PORT

10023

TCP console port (PuTTY auto-launch follows it). Give each instance its own port.

EMULATR_FLASH_ROM

ds10_flash.rom

Backing file for the flash / NVRAM image. Isolate it per instance to avoid corruption.

EMULATR_PLATFORM_CONFIG

derived DS10

Override the platform manifest.

 

Logging / diagnostics and device traces

Variable

Effect

EMULATR_CHIPSET_DIAG_OFF

Silences the chipset CSR diagnostics (inverted sense: setting it turns them OFF).

EMULATR_TRACE_WINDOW

Forces the retire-compact trace window.

EMULATR_RETIRE_TRACE_DIR

Output directory for retire / breakpoint / profiler trace files.

EMULATR_IIC_TRACE

Trace PCF8584 IIC bus transactions.

EMULATR_FLASH_TRACE

Trace FlashRom command / erase / write activity.

EMULATR_PIC_TRACE

Trace the 8259 PIC pair (IRQ assert / EOI).

 

Probe / experimental

EMULATR_IDLEWARP fast-forwards the idle loop to the next timer tick (clean). EMULATR_GCT_WATCH, EMULATR_BREAK_ON_PA10, EMULATR_CHECKPOINTS, and EMULATR_GATE are pinpoint debug gates. Note: EMULATR_RSCCWARP is QUARANTINED (caused boot corruption) and EMULATR_TICKWARP is a deprecated no-op.

 

Compile-time flags (NOT runtime env vars)

These are CMake options baked in at build time -- setting them in the shell does nothing: EMULATR_MEMDIAG (per-instruction logging, ~900x slowdown, keep OFF), EMULATR_DIAGNOSTIC_LOGGING (non-Release per-subsystem logging), and the EMULATR_LOG_* spdlog level macros.

 

CLI flags that mirror env vars

--autosnapshot off = EMULATR_AUTOSNAP=off; --no-autoload = EMULATR_NO_AUTOLOAD=1. --max-cycles, --snapshot-on-pc, and --snapshot-name-tag have no env equivalent.
