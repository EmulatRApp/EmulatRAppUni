ISO-9660 PFD (CD001)

An ISO-9660 image begins with a 16-block (32 KiB) system area, after which LBA 16 holds the Primary Volume Descriptor (PVD). The firmware's first coherence check is the standard identifier "CD001" at offset 1 of that descriptor. VirtualIsoDevice serves the PVD so a READ(10) of LBA 16 returns it. The doctest that locks this behavior:

 

TEST_CASE("VirtualIsoDevice: serves the ISO-9660 PVD at LBA 16 (CD001)") {

 // System area = 16 zero blocks; LBA 16 = PVD.

 std::vector<uint8_t> img(17 * 2048, 0);

 uint8_t* pvd = img.data() + 16 * 2048;

 pvd[0] = 0x01; // descriptor type = PVD

 std::memcpy(pvd + 1, "CD001", 5); // standard identifier

 pvd[6] = 0x01; // version

 // block size 2048 (both-endian) at offset 128

 VirtualIsoDevice cd; REQUIRE(cd.loadMedia("pvd.iso"));

 uint8_t buf[2048]; ScsiCommand c = read10(16, 1, buf);

 cd.handleCommand(c);

 CHECK(c.status == ScsiStatus::Good);

 CHECK(std::memcmp(buf + 1, "CD001", 5) == 0);

}
