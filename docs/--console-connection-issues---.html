Console Connection Issues

For refused, timed-out, or silent console connections, the full client coverage is in the Terminal Clients page. In brief, check three things:

The listener is up -- the log shows "Listening on TCP port ...".

The port matches your client (the default is 10023; an instance may have been started on another with EMULATR_CONSOLE_PORT).

The client is in RAW mode -- the socket is raw, with no Telnet option negotiation.
