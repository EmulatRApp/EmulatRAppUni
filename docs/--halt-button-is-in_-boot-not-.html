Halt Button is IN, BOOT NOT POSSIBLE

Symptom

At power-up the console prints "Halt Button is IN, AUTO_ACTION ignored", and at the >>> prompt any boot attempt (for example b dqa1) is refused with "Halt Button is IN, BOOT NOT POSSIBLE".

 

What it means

The firmware reads the front-panel RUN/STOP (halt) switch and finds it asserted (IN). A halted machine refuses to boot and ignores auto_action -- on real hardware you would release the halt switch to allow a boot.

 

In EmulatR

This halt gate is under active investigation, and it is the current reason a boot at >>> does not yet proceed. The exact source the firmware tests for the halt condition is being pinned with an armed hardware trace before any fix is written. The e pmem:80130000FF8 read below returning 7FFFFFFFFFFFFFFF is part of that diagnostic (it arms the trace). A representative session:

 

AlphaServer DS10 268 MHz Console V7.3-2, Feb 27 2007 13:25:53

Halt Button is IN, AUTO_ACTION ignored

>>> e pmem:80130000FF8

pmem: 80130000FF8 7FFFFFFFFFFFFFFF

>>> b dqa1

Halt Button is IN, BOOT NOT POSSIBLE

>>>

 

Until the halt source is modeled, this message on a boot attempt is expected. Progress is tracked in Current Development Status.
