Executing

Beyond the basic launch (see Launching EmulatR), a few run scenarios come up often. All are launched from the Emulatr root and write diagnostics to a redirected log.

 

Traced boot

Turn on the device trace channels and put the console on an alternate port (so it coexists with another instance). PuTTY follows EMULATR_CONSOLE_PORT.

 

EMULATR_NO_AUTOLOAD=1 EMULATR_CONSOLE_PORT=10024 \

 EMULATR_IDE_TRACE=1 EMULATR_IIC_TRACE=1 EMULATR_FLASH_TRACE=1 \

 ./out/build/release/Emulatr.exe \

 --firmware firmware/ds10_v7_3.exe --autosnapshot off 2> run_ide.log

 

Unattended (overnight) run

Use nohup so the run survives logoff; leave autosnapshot on and mirror the console to the log.

nohup env EMULATR_NO_AUTOLOAD=1 EMULATR_CONSOLE_SNAPSHOT=1 \

 EMULATR_CONSOLE_MIRROR=1 EMULATR_CONSOLE_PORT=10023 \

 ./out/build/relwithdebinfo/Emulatr.exe --firmware firmware/ds10_v7_3.exe \

 > run_overnight.log 2>&1 &

 

Harvest the slowest console phases

After a run with the console mirrored to the log, the slowest timestamped phases fall out of the log with:

grep -a '\[CON' boot.log \

 | sed -E 's/.*\[CON [^+]*\+ *([0-9.]+) ms\] ?/\1\t/' \

 | sort -rn | head -25
