Current Platform Support

A platform manifest is selected from the model name as lower(model) + "_platform.json" (for example, DS10 gives ds10_platform.json). The table below summarizes current status. "Validated" means the platform has been driven to the SRM >>> prompt in EmulatR.

ES45

es45_platform.json

EV67 / 21272

Manifest present -- not yet validated

 

A missing platform file is a hard error -- there is no silent fallback. The selection order is: EMULATR_PLATFORM_CONFIG, then a platformConfigPath ini key, then the derived name.
