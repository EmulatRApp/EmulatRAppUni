Current Development Status

This page is a plain-language snapshot of where EmulatR stands as of June 2026 -- what works end to end, the milestone the project is working toward next, and the known rough edges. It is deliberately a moving target; the authoritative day-to-day record lives in the engineering journals and the dated CHECKPOINT notes. For a per-subsystem breakdown see the Feature Matrix.

 

What works today

A clean cold boot runs the original, unmodified DEC SRM console firmware all the way to the interactive >>> prompt, fully usable over a serial terminal. The commands show config, show memory, show pal and show version all return correct output. Firmware is a real SRM 7.3-1 image carried in emulated flash; the LFU update srm path works, and machine-state snapshots have been captured for both the firmware-update (UPD>) and console (>>>) states.

 

The current frontier: getting to an OS boot

The next major milestone is booting an operating system. Two concrete gaps stand between the prompt and that goal:

No bootable disk is enumerated yet. At the prompt, show device lists only the floppy (dva0) -- there is no dka SCSI disk to boot from. Disk-image support and the storage backing exist (see Virtual Disk Images); the gap is the device model and enumeration that would make a disk appear to the firmware.

On-board PCI devices are not walked. show config reports only the Cypress 82C693 bridge on PCI Hose 00 -- the on-board Ethernet and SCSI controllers are not yet modeled or enumerated, so the firmware cannot find them.

 

A separate active investigation concerns the front-panel halt gate: b dqa1 currently reports "BOOT NOT POSSIBLE". This is being diagnosed with an armed hardware trace to identify the exact source the firmware tests before any fix is written -- an evidence-first approach rather than guessing at a register.

 

Known cosmetic issues

Two harmless display quirks are tracked: show device prints each line twice (console line-doubling, also seen at UPD>), and the SROM revision shown by display / show config contains one garbled byte. Neither affects operation.

 

What is planned next

• SRM environment persistence -- make set at the prompt survive a reset and cold boot, like real NVRAM.

• PCI bus enumeration with on-board device models (Ethernet, SCSI), so the firmware discovers real hardware instead of an all-ones placeholder.

• A SCSI disk plus boot media so a dka device exists to boot, leading into an OS install from CD-ROM.

 

Snapshot date: June 2026. Because status changes frequently, treat this page as orientation and confirm specifics against the current build and the latest CHECKPOINT journal.
