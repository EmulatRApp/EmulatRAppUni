Diagnostics & Debugging

Diagnostics are turned on with runtime EMULATR_* environment variables, set in the launching shell before the process starts. The complete table lives in the Reference section's Environment Variables page; this page groups them by what they do.

 

Three gotchas to know first

•Presence, not value. Most gates test only that the variable is set, so VAR=0 still ENABLES them. The few that parse a value (the autosnapshot / marker / path / numeric knobs) are noted in the reference.

•Read once. The trace and probe gates cache their result on first call, so they are effectively read at process start; exporting them mid-run has no effect.

•Paths are CWD-relative. Launch from the Emulatr root so relative output and snapshot paths resolve to the canonical set.

 

Device / subsystem traces

Presence-gated, read once, written to stderr: EMULATR_IIC_TRACE (PCF8584 IIC bus transactions), EMULATR_FLASH_TRACE (FlashRom command / erase / write activity), EMULATR_PIC_TRACE (the 8259 PIC pair -- IRQ assert / EOI). Redirect to per-trace logs, for example 2> iic.log.

 

Chipset diagnostics and retire trace

The chipset CSR diagnostic stream is on by default; EMULATR_CHIPSET_DIAG_OFF silences it (note the inverted sense -- setting it turns diagnostics OFF). EMULATR_TRACE_WINDOW forces the retire-compact trace window, and EMULATR_RETIRE_TRACE_DIR sets the output directory for the retire / breakpoint / profiler trace files (see SRM Performance Logging).

 

Probe and break gates

For pinpoint debugging: EMULATR_GCT_WATCH (store-watch on the GCT / FRU config-tree region), EMULATR_BREAK_ON_PA10 (conditional break on accesses to PA 0x10..0x17), EMULATR_CHECKPOINTS (named PC checkpoints), and EMULATR_GATE (a PC / condition gate controlling trace windowing).

 

Compile-time diagnostics (NOT runtime env vars)

Some EMULATR_* names are CMake options baked in at build time -- setting them in the shell does nothing. The important one is EMULATR_MEMDIAG: a per-retired-instruction fprintf+fflush that imposes roughly a 900x slowdown and must stay OFF for normal use. EMULATR_DIAGNOSTIC_LOGGING (centralized per-subsystem logging, non-Release builds) and the EMULATR_LOG_* spdlog level macros are also build-time.
