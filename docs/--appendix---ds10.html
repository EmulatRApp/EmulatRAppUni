Appendix - DS10

Reference material specific to the DS10 platform -- the only model EmulatR currently models and drives end to end. This page summarizes the hardware EmulatR presents and the device manifest that defines it; the sub-topics carry the environment-variable reference, the SRM and SuperIO analysis, and the engineering status.

 

Hardware summary

The AlphaServer DS10 is a single-processor Tsunami-family workstation. EmulatR presents a single 21264 (EV6) CPU, the 21272 (Tsunami) core-logic chipset, and a Cypress CY82C693 combination ISA bridge / IDE controller / SuperIO. The default configuration reports 1024 MB of memory and a 268 MHz cycle clock. The platform is defined declaratively by ds10_platform.json; a missing manifest is a hard startup error with no fallback.

The IDE controller carries two storage units on its primary channel: dqa0 (primary master) is the bootable ATA fixed disk (raw flat image, 512-byte sectors, byte offset = LBA x 512), and dqa1 (primary slave) is the ATAPI CD. Media for each is resolved from the manifest against the [Storage] diskDir in the ini; an empty media string means no disk / no disc.

 

IIC / FRU devices

The DS10 manifest also models the IIC (I2C) bus the firmware probes during GCT/FRU initialization: two system-status devices (0x70, 0x72), an SMB FRU EEPROM for the DS10 board (0xA2) and one for the EV6 CPU (0xA4), and a 256-byte RCM NVRAM (0xC0). The PCI vendor/device IDs and BAR sizes are marked provisional in the manifest pending a real hardware dump.
