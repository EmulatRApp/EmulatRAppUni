1. PURPOSE

  One map of every physical-address window, software seam, and device model
  in V4 -- existing, stubbed, and missing -- each with its authority
  citation.  Goals:

    (a) "What absorbs this address?" becomes a table lookup, never a hunt
        (the 2026-06-03 IIC stall hid for weeks behind a silent absorber).
    (b) New device work (IIC, PCI walk, NIC, SCSI) is scheduled against an
        explicit gap list instead of discovered by the next boot stall.
    (c) Audit rule: every UNHANDLED log line and every silent float maps
        to a row here, or the map is wrong.

2. ROUTING MACHINERY (the call chain)

  CPU LD/ST (MemDrainer / Ev6Translator, VA -> PA)
    -> GuestMemory / ISystemBus
    -> TsunamiChipset -- the SOLE PA decoder (arbiter model, landed
       2026-05-22; GuestMemory is a dumb store).  Decode order:

       1. PAL scratch carve-out   PA 0xFFF.FF00.0000 + 1 MB (top of 16 TB)
          (TsunamiChipset.h kPalScratchBase) -- PALcode-private spill;
          decoded ahead of everything.
       2. DRAM                    isDramAddress() -> GuestMemory pages.
       3. TIG flash window        PA 0x801.0000.0000, byte at 64-byte
          stride, 128 MB extent (2 MB device x 64) -> FlashRom.
          Authority: firmware xtig() mapping; AXPBox registration.
       4. Chipset CSR windows     (sec. 3 table) -> Cchip / Dchip / Pchip
          CSR handlers (Tsunami21272_RegisterMap.h).
       5. Pchip dense I/O         per-port registry (IoPortRange list)
          -> IIoPortHandler; UNREGISTERED ports -> Cypress fallback
          (setIoPortHandler) which floats 0xFF reads / drops writes
          SILENTLY.  <-- AUDIT ITEM, sec. 7.
       6. Pchip config space      (CfgType0) -> registerPciDevice map
          -> IPciDeviceHandler (config reads/writes only).
       7. Pchip PCI MEMORY space  NO CLAIMANT REGISTRY EXISTS.  Reads and
          writes fall through to "TsunamiPchip: UNHANDLED OUTER
          READ/WRITE".  <-- GAP G3, sec. 6; the IIC controller lives here.
       8. Else: NXM -- asynchronous per HRM 6-29: bus ReadDataError +
          MISC<NXM> sticky latch + b_irq<0> (landed 2026-05-28; never a
          synchronous MCHK).

  Interrupt fabric (delivery side):
    Cchip DIM0..3 / DRIR / MISC<ITINTR> latches; b_irq<0..3> per CPU;
    interval timer FIRE (cycle-edge latch, intervalTimerShouldFire) is
    decoupled from DELIVER (canAcceptInterrupt(22): relocation done,
    palBase set, IER<EIEN2>, not PALmode) -> stageInterruptDivert
    (Machine.cpp) -> PAL INTERRUPT entry palBase+0x680.
    Authority: HRM ch. 6 (Cchip), ch. 10 CSRs; ISA-bridge interrupt is
    DRIR<55> (PC264 pci_irq_table -- memory pc264-pci-irq map).

  Console path (host seam):
    Uart16550 (deviceLib/Tsunami/Uart16550.h) THR/LSR at COM1 ->
    ConsoleManager backend -> SRMConsoleDevice (TCP, PuTTY autolaunch)
    + stderr mirror.  Guest-visible state must stay host-independent
    (determinism rule); RX input path is gap G2.

6. GAP LIST (ranked for the path to P00>>>)

  G1  PCF8584 IIC stub at PCI MEM 0xFFFF0000-1.  ACTIVE BLOCKER: powerup
      loops 2 s/lap on LAB=1 decoded from the Pchip unhandled-PCI-mem
      read float.  Spec complete (IIC_PCF8584_Specification.txt); depends
      on G3-lite.
  G2  UART RX input path (COM1 receive + LSR DR + interrupt or poll).
      Required AT the prompt: yy_reset/read_with_prompt poll for input.
      Without it the prompt prints but cannot hear.
  G3  PCI MEM claim seam in TsunamiPchip (fixed-range registration is
      enough for G1; BAR-tracked claims arrive with the PCI walk).
      G3-lite (fixed range) is a prerequisite of G1 and ~30 lines.
  G4  PCI bus walk + BAR machinery (USPO-PCI ch. 6) -- silences residual
      probe noise; prerequisite for NIC/SCSI.  Non-fatal today.
  G5  DE500/21143 NIC model -- after G4.
  G6  SCSI fabric port from V1 -- after G4; the `>>>` boot-device work.
  G7  8042 keyboard input (only if serial console proves insufficient
      for the prompt; SRM falls back to COM1).
  G8  TIGbus control registers (2026-06-03).  TIGbus is a peripheral bus,
      not flash-only; firmware drives control regs at high offsets:
      0xE00004 = system reset (console_restart), 0xC00028+id = per-CPU
      start (start_secondary).  Today these fall in the unmodeled gap
      between the flash extent and Cchip CSR -> halt.  NOTE: this is
      DOWNSTREAM of the real blocker -- console_restart only runs because
      build_dsrdb's fread already SYSFAULTed (see frontier note below).
      A TIGbus control-register model (reset = restart-or-noop, CPU-start
      = absorb for the single-CPU build) silences the symptom; the fread
      SYSFAULT is the cause to fix first.

7. AUDIT RULES (the "sealing" discipline)

  A. No silent absorbers.  The Cypress ISA fallback and the Pchip
     PCI-MEM fallthrough must each log unknown addresses ONCE per unique
     address (throttled), so the next unmodeled device costs minutes,
     not weeks.  (The IIC stall hid behind exactly this.)
  B. Every claimed range cites an authority document in this map; every
     row names its V4 file; every UNHANDLED line in a boot log must match
     a MISSING row here -- if it does not, the map gets a new row first,
     the code second.
  C. Status transitions (MISSING -> STUB -> LIVE) happen via a spec
     document per house process (Toy Specification and
     IIC_PCF8584_Specification.txt are the precedents).
  D. Width discipline: byte-lane devices float 0xFF on wide reads and
     drop wide writes with a one-shot note (ToyRtc precedent).
  E. Determinism: no device may consult host time or host I/O readiness
     for guest-visible state (ToyRtc cyclesPerSecond pattern; UART RX
     will need an injection-queue design honoring this).
