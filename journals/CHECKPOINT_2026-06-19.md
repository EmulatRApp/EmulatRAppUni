- **Done since last checkpoint:**
  - **Core finding nailed down:** `m_cpu.cycleCount` is doing **two jobs at once** — it is both CPU0's architectural PCC **and** the system timebase that the RTC, interval timer, flash-flush, snapshot cadence, and IDLEWARP all read. With one CPU the two numbers coincide so the conflation is invisible; with a 2nd `CpuState`, reading CPU0's PCC as "the system clock" is the **CPU0-as-sink bug in disguise**. This is the load-bearing insight the whole phase is built on.
  - **Structural move specified:** split the current fused `stepCycle` into (1) `cpuKernel(cpu)` — per-agent: interrupt poll + `PipelineDriver::step` for one CPU, **no global side effects**; and (2) `systemTick(now)` — once per quantum, dispatcher-level: stop-sentinel, interval-timer fire+divert, flash flush, snapshot cadence, IDLEWARP, all keyed off the **logical clock** (the `Dispatcher`'s existing `LogicalClock`, not PCC). This split is what kills the "single-agent-by-shape" double-fire and is the real prerequisite for a 2nd CPU.
